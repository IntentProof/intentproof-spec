# v0.1 local contract

Normative summary for IntentProof **v0.1.0** (local-first OSS). Hosted
control-plane APIs and tenant lifecycle are out of scope.

Canonical copy also lives in
[`intentproof-tools`](https://github.com/IntentProof/intentproof-tools/blob/main/docs/v0.1-local-contract.md).

## Bundle archive (`.proof.tar.zst`)

| Member | Required | Role |
|--------|----------|------|
| `manifest.json` | yes | File SHA-256 list, Merkle roots, optional verification profile |
| `flow.json` | yes | Flow metadata |
| `events.jsonl` | yes | Signed execution events |
| `policy.json` | yes | Policy document (may include `policy_fingerprint`) |
| `run.json` | yes | Bundled verification run |
| `attestations.jsonl` | no | External attestations |
| `certificate.json` | no | Platform certificate when present |
| `keys/*.pub` | no | Verification keys shipped with bundle |

Format: zstd-compressed tar. Plain tar is accepted in unit tests only.

## Offline verification

### Human output (`intentproof-verify`)

| Command | Behavior | Exit |
|---------|----------|------|
| `intentproof-verify <bundle>` | Structural integrity | 0 pass, 1 fail |
| `intentproof-verify explain <bundle>` | Integrity + policy replay summary | 0 pass, 1 fail |
| `intentproof-verify replay <bundle>` | Integrity, then fresh policy eval (JSON run on stdout) | 0 pass, 1 fail |

Passing structural verify prints a first line of the form `✓ pass: bundle.verify_pass`.

### Machine output (`intentproof verify`)

`intentproof verify <bundle>` prints JSON `VerifyResult` (`status`, `reason`,
`findings`) on stdout with the same integrity semantics as above.

## Policy replay

- Replay merges `events.jsonl` into the flow object and evaluates `policy.json`
  using the Go evaluator shipped in `intentproof-tools` (`pkg/verifier`).
- Counterparties must run structural verify before trusting replay output.
- `explain` compares replay status to bundled `run.json` when present.

## Reason catalog

Stable codes: `semantics/reasons.json`. `explain` loads catalog copy when
`INTENTPROOF_SPEC_DIR` points at this repository.

## Compatibility and versioning

- **v0.1.0** tags this contract. Breaking changes to manifest layout, hash
  rules, canonical JSON, or verifier exit semantics require a new minor version.
- SDKs should document which spec ref / tools release they were tested against.
